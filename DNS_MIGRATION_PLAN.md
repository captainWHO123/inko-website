## 📋 从截图中观察到的 DNS 记录

根据你的 Cloudflare DNS Records 页面，我需要确认以下信息：

### 关键记录类型：
1. **A 记录** - 指向服务器 IP 地址
2. **CNAME 记录** - 指向其他域名
3. **MX 记录** - 邮件服务器
4. **TXT 记录** - 验证和其他信息

## 🎯 迁移步骤

### 第 1 步：部署新项目到 Cloudflare Pages

1. **创建新的 Pages 项目**
   - 项目名称：`inko-website`
   - 连接到 GitHub 的 `inko-website` 仓库
   - 配置：
     - Framework: Vite
     - Build command: npm run build
     - Output directory: dist

2. **获得临时的 .pages.dev URL**
   - 部署成功后，你会得到类似：
   - `https://inko-website.pages.dev`

### 第 2 步：测试新项目

1. **访问你的 .pages.dev URL**
2. **确认网站正常显示**
3. **测试 Kit 表单功能**

### 第 3 步：更新 DNS 记录

#### 如果你想完全替换旧网站：

**方法 A: 删除旧的 A/CNAME 记录，添加新的**

1. **在 DNS Records 页面找到旧网站的记录**
   - 通常是 `A` 记录（指向服务器 IP）
   - 或 `CNAME` 记录（指向其他域名）
   - 名称可能是 `@`、`www` 或其他子域名

2. **删除旧记录**
   - 点击记录右侧的删除按钮
   - 或先编辑，记下设置后再删除

3. **添加新的 CNAME 记录**
   - Type: `CNAME`
   - Name: `@` (主域名) 或 `www` (子域名)
   - Target: `inko-website.pages.dev`
   - Proxy status: Proxied (橙色云朵)

**方法 B: 暂时保留旧记录，添加新记录**

1. **添加新的 CNAME 记录**
   - Type: `CNAME`
   - Name: `new` (或其他测试子域名)
   - Target: `inko-website.pages.dev`
   - Proxy status: Proxied

2. **测试新版本**
   - 访问 `new.yourdomain.com`
   - 确认新网站正常工作

3. **确认后切换主域名**
   - 删除旧的主域名记录
   - 添加新的主域名记录

## ⚠️ 重要注意事项

### 关于 DNS 切换：

1. **删除 DNS 记录要谨慎**
   - 一旦删除，旧网站立即下线
   - 建议先添加新记录，再删除旧记录

2. **DNS 传播时间**
   - 通常需要 5-10 分钟
   - 最长可能需要 24-48 小时
   - 使用 dnschecker.org 检查传播状态

3. **保持旧网站可用**
   - 在确认新网站完全正常前
   - 保持旧网站运行
   - 可以使用子域名测试
